# Hermes Runtime — RBAC

Модуль не имеет публичных эндпоинтов и собственной авторизации. Доступ — опосредованный, через [Agent Proxy](../agent-proxy/06-rbac.md) (клиентская auth `X-API-Key` + `X-User-Id`, [ADR-044](../../adr/ADR-044-client-api-key-auth.md)).

- **Субъект операций** — `user_id`, переданный потребителем (= `X-User-Id` субъекта запроса Agent Proxy). Модуль оперирует ровно этим `user_id`; не принимает «чужой» `user_id` из недоверенного источника.
- **Изоляция владельца** — все операции registry скоупятся по `user_id` (PK таблицы); невозможно адресовать чужой инстанс.
- **Admin** — модуль не вызывается из admin-контура напрямую. (Operational deprovision/diagnostics операторами — потенциальное будущее admin-расширение, не в этой поставке.)
- **Reaper** — системная задача (`lifespan`), не пользовательский принципал; оперирует всеми инстансами по `status`/`last_active_at`, без привязки к запросу.
