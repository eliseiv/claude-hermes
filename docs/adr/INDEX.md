## Ревизии

- **ADR-046 (расширение/частичная ревизия 2026-06-24 → [ADR-055](ADR-055-hermes-instance-llm-config-contract.md)):** реальный e2e вскрыл интеграционный дефект конфигурации LLM Hermes-инстанса (401 «Provider: openrouter, Model: (пусто)»). [ADR-055](ADR-055-hermes-instance-llm-config-contract.md) уточняет §1 (env-проброс: убрать игнорируемый `LLM_MODEL` и `LLM_PROVIDER`) и §5/§6 (рендер `config.yaml`: добавить секцию `model.default`/`model.provider`[/`base_url`], запретить `provider:auto`, fail-fast валидация провайдера/модели). Тело ADR-046 не переписано (immutability); контракт `config.yaml`/env читается из ADR-055.
- **ADR-054 (расширение 2026-06-24 — §2a Continuation tool-loop под claimed trial):** при реализации [TD-006](../100-known-tech-debt.md) выявлен не покрытый §2 кейс на пути `/v1/chat/tool-result`: trial-ход с **client-side** инструментами возвращает `tool_call` (claim уже выполнен на `/chat/run`), и continuation после закрытия барьера ошибочно блокируется `blockReason=trial_used` своим же claim'ом (re-evaluate видит `trial_used=TRUE`; регрессия claim-before). Добавлен §2a: детерминированный признак in-progress trial-хода (`mode=credits ∧ subscription=none ∧ trial_used=TRUE ∧ assistant_tool_step_id(message_step_id)≠None`) — решается **в orchestrator** (policy остаётся чистой функцией); не-блокировка continuation своего trial-хода по `trial_used`; reconcile-откат при неуспехе continuation тем же признаком (замена request-scoped `claimed_trial`, который не переносится между запросами). Без миграции, публичный контракт `/chat/tool-result` не меняется. Тело §1–§4 не переписано (расширение добавлением раздела). Синхронно обновлён [modules/chat-orchestrator/03-architecture.md §Поток /v1/chat/tool-result](../modules/chat-orchestrator/03-architecture.md).
